# AI 全自动化交付

## 定义

AI 全自动化交付（AI Automated Delivery，L3）是指通过多 AI Agent 自主协同，实现**从需求到部署发布的端到端无人值守自动化交付**，不需要人工在各环节之间切换。与人机协同（L2）的本质区别在于：AI 不仅能生成代码，还能处理需求理解、技术方案、测试执行、部署发布等所有中间环节。

## 三阶段演进路线

| 阶段 | 名称 | 描述 |
| --- | --- | --- |
| L1 | 纯人工 | 所有环节由人完成 |
| L2 | 人机协同 | AI 辅助编码，人工参与决策节点 |
| **L3** | **AI 全自动** | 需求到部署无人值守，AI Agent 自主协同 |

L3 成熟标志：形成"N Agent 交付模式"，多个 AI 智能体自主协同完成完整交付。

## 核心公式

> **AI 全自动化交付 = LLM + Harness Engineering（工作流程 + 知识库 + Skills 管理）+ Business Engineering（业务专属 Skills + 知识库）**

各部分说明：
- **LLM**：大语言模型作为推理引擎
- **Harness Engineering**：工作流程编排 + 高质量知识库 + 可复用 Skills 体系
- **Business Engineering**：特定业务领域的专属能力积累

## 实现路径

腾讯审核团队的三段式能力阶梯（当前在 L2 推进中）：

1. 技术方案 → 代码（已成熟）
2. 技术方案 → 交付（测试+部署打通）
3. 需求 → 交付（端到端全链路，L3 入口）

### 关键工程要素

**标准化交付流程**：通过流程编排 Skill 将需求、研发、测试、部署串联，出现问题自动回滚

**需求标准化**：
- 标准需求模板（需求概述、正文、版本管理、外部依赖）
- 需求澄清规则与评分规则
- PRD-Agent：支持产品在 AI Agent 上进行需求编写和澄清

**知识库**：
- 业务知识库（为 AI 生成技术方案提供领域上下文）
- 代码知识库（为 AI 生成高质量代码提供结构了解）

**质量门禁**：需求评审 → 方案评审 → 代码 CR → 安全检测，严格阻塞式流程

### 双框架体系

**交付框架**（效率）：全流程自动化串联 + 闭环反馈与自修复

**治理框架**（质量）：运行时日志 + 监测指标 → 自动巡检、异常感知、知识自动回溯更新

> "交付驱动效率，治理保障质量"的双轮驱动体系

## 效果数据（腾讯审核团队，6 个试点需求）

- 需求评分：80 分以上
- 技术方案评分：80 分以上
- 平均生成对话轮次：2 轮
- 代码行采纳率：90%+
- AI 生成率：80%+

## 与 [[Harness Engineering]] 的关系

AI 全自动化交付（L3）必须以 [[Harness Engineering]] 为支撑——没有形式化约束环境、没有质量闸门、没有反馈闭环，全自动化只会加速错误扩散。[[Harness Engineering]] 是 L3 从"可能"变成"可信"的关键。
