# 白家杰

## 基本信息

- **领域**：AI 辅助工程开发、Harness Engineering 实践
- **已知项目**：JK Launcher（面向 Unity 研发流程的 Windows 桌面启动器工具）
- **主要贡献**：从零搭建 Harness Engineering 的完整工程实践手册，提出六大零件分类法和七 Agent 架构演化路径

## 主要贡献

### Harness Engineering 工程落地实践

白家杰以 JK Launcher 为真实案例，系统记录了如何从零搭建 [[Harness Engineering]]。核心洞察：

- **六大零件分类**：Rule（软约束）/ Skill（标准操作手册）/ Sub Agent（角色分工）/ Workflow（接力赛规则）/ Scripts（硬门禁）/ MCP（外接接口）
- **SPEC 先行原则**：第一步不是写 Rule，而是与 AI 反复磨透设计规格文档
- **结构化调度的选择**：明确放弃去中心化协作，选择固定角色+固定流程，因为"真正贵的不是 token，真正贵的是失控"

### 七 Agent 架构

被真实问题逼出来的七 Agent 结构：PM（总路由器）/ 需求分析 / 方案设计 / 闸门总控 / 开发实现 / 代码审查 / 测试验证。其中 PM 严格限制为"总路由器"，不越界给专业意见。

### Harness 四块拼图

提出 Harness Engineering 的四块拼图框架：约束与流程 / 反馈 / 知识库（索引）/ 进化。
