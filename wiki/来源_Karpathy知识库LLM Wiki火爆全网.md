# Karpathy 知识库「LLM Wiki」火爆全网

## 基本信息

- **来源**：知乎 / 机器之心
- **发布日期**：2026-04-05
- **主要人物**：[[Andrej Karpathy]]

## 核心要点

1. [[Andrej Karpathy]] 以 gist 形式发布了 [[LLM Wiki]] 的构建思路，引发全网热议
2. 核心理念：LLM 不是每次从零检索（[[RAG]]），而是**增量构建并维护一个持久化 wiki**
3. 系统分为三层架构：原始数据层、Wiki 层、Schema 层
4. 被社区视为一种[[元框架]]（meta-framework），不依赖特定模型或技术栈
5. 延伸方向：通过合成数据与微调，将知识内化进模型权重

## 详细摘要

### 背景

[[Andrej Karpathy]] 近期分享了一种新的知识管理模式——[[LLM Wiki]]。他将想法整理成 gist 形式发布，提出在 Agent 时代，我们已经不需要分享具体代码或应用，只需分享「想法」，让对方的 Agent 自动完成定制和实现。

gist 地址：`https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f`

### 与 RAG 的区别

传统 [[RAG]] 模式下，每次提问 LLM 都从零开始检索和拼接。[[LLM Wiki]] 的不同之处在于：

> 当你添加新的资料时，模型不只是简单地索引以备后用，而是会真正去阅读它，提取关键信息，并将其整合进现有的 wiki：更新实体页面、修订主题总结、标记新信息与旧结论之间的冲突。

知识被编译一次，并持续更新，而不是每次查询时重新推导。

### 三层架构

1. **原始数据层**：文章、论文、图片等，不可变，LLM 只读不写
2. **Wiki 层**：LLM 生成的 Markdown 文件集合，包含摘要、实体页、概念页、对比分析等
3. **Schema 层**：指导性文档（如 CLAUDE.md），让 LLM 从通用聊天模型变为有纪律的 wiki 维护者

### 三大操作

- **摄入（Ingest）**：处理新资料，一个来源往往影响 10-15 个 wiki 页面
- **查询（Query）**：围绕 wiki 提问，好的回答可归档为新页面，实现知识复利
- **质量检查（Lint）**：定期健康检查，发现矛盾、补全缺失、挖掘新方向

### 社区评价

- 被认为是一种[[元框架]]，具备长期稳定性和适用性
- 中等规模下（~100篇文章、~40万字）不依赖传统 RAG，索引+摘要即可支撑检索推理
- 延伸方向：通过合成数据+微调将知识内化进模型权重，逼近「自我增强的知识系统」

### 实际使用方式

Karpathy 的工作方式：一边打开 LLM Agent，一边打开 [[Obsidian]]——模型编辑内容，用户实时浏览结果。

> Obsidian 是 IDE，LLM 是程序员，wiki 是代码库。

### 应用场景

- 个人成长：目标、健康、心理、自我提升
- 研究：论文、文章、报告的综合知识体系
- 读书：章节整理、人物/主题/情节关联
- 企业/团队：接入 Slack、会议记录、项目文档
- 竞品分析、尽职调查、旅行规划、课程笔记等
