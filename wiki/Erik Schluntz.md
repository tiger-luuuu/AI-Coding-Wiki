# Erik Schluntz

## 基本信息

- **职位**：研究员，编程智能体方向负责人
- **著作**：《构建高效智能体》（合著）
- **已知贡献**：Vibe Coding 生产实践方法论、叶子节点策略、可验证抽象层理念

## 主要贡献

### 生产环境 Vibe Coding 方法论

Schluntz 因手臂骨折被迫全权委托 Claude 处理工作，由此形成了在生产环境中负责任地进行 [[Vibe Coding]] 的系统性方法：

- **叶子节点策略**：将 AI 生成的代码限制在代码库中不被其他模块依赖的末端功能，主干架构保持人工控制
- **可验证抽象层**：开发者需找到无需阅读底层代码即可验证正确性的抽象层（验收测试、E2E 测试等）
- **AI 的 PM 工作流**：前置 15-20 分钟规划（探索代码库 → 制定计划 → 汇总上下文），再让 AI 执行

### 22000 行代码生产合并案例

在 Anthropic 内部强化学习代码库中，主导了一次 22000 行代码的合并，原本需要两周的工程压缩到 1 天，四大策略：深度 PM 引导 + 叶子节点限制 + 核心人工审查 + 可验证检查点。

### TDD 与 Compact 实践

- 极简 E2E 测试原则（3 个测试）
- 上下文压缩（Compact）在自然停顿点执行，先规划写文档再压缩
