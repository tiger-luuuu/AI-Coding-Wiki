# LLM Wiki — Schema

## 项目概述

这是一个基于 LLM 维护的个人知识库，用于研究和学习。LLM 负责摄入原始资料、生成并维护 wiki 页面、保持交叉引用的一致性。用户负责提供资料来源、提出问题、引导分析方向。

使用 Obsidian 浏览 wiki，使用 Claude Code 作为 LLM 代理。

## 语言约定

- 所有 wiki 页面内容使用**中文**
- 文件名使用中文（与 Obsidian 页面标题一致）
- 标签（tags）使用中文
- 如果原始资料是英文，摘要和 wiki 内容仍用中文撰写，但保留关键术语的英文原文（用括号标注）

## 目录结构

```
raw/            # 原始资料（只读，LLM 不修改）
  assets/       # 图片、附件
wiki/           # LLM 生成和维护的 wiki 页面
  index.md      # 内容索引
  log.md        # 操作日志
```

## 页面类型

### 来源摘要页

对应 `raw/` 中的每份资料。文件名格式：`来源_<简短标题>.md`

```yaml
---
类型: 来源摘要
来源文件: "raw/xxx.md"
日期: 2026-04-07
标签: [标签1, 标签2]
---
```

内容包含：
- 基本信息（作者、发布日期、来源链接等）
- 核心要点（3-5 个关键观点）
- 详细摘要
- 与 wiki 中其他页面的关联（使用 `[[双链]]`）
- 引用的原文关键段落

### 概念页

核心概念、理论、方法论的独立页面。文件名即概念名。

```yaml
---
类型: 概念
来源数: 3
最后更新: 2026-04-07
标签: [标签1, 标签2]
---
```

内容包含：
- 定义
- 详细说明
- 相关概念（`[[双链]]`）
- 来源引用

### 实体页

人物、组织、工具、项目等。文件名即实体名。

```yaml
---
类型: 实体
实体类别: 人物/组织/工具/项目
最后更新: 2026-04-07
标签: [标签1, 标签2]
---
```

### 主题页

跨多个来源的综合分析、综述、演变追踪。文件名格式：`主题_<标题>.md`

```yaml
---
类型: 主题
来源数: 5
最后更新: 2026-04-07
标签: [标签1, 标签2]
---
```

### 比较页

对比分析。文件名格式：`比较_<A>与<B>.md`

```yaml
---
类型: 比较
最后更新: 2026-04-07
标签: [标签1, 标签2]
---
```

## 页面格式约定

1. **所有页面必须有 YAML frontmatter**，至少包含 `类型`、`标签`、`最后更新`
2. **使用 Obsidian 双链语法** `[[页面名]]` 进行交叉引用
3. **标题层级**：页面标题用 `#`，主要章节用 `##`，子章节用 `###`
4. **引用原文**用 `>` 引用块，标注来源页面链接
5. **矛盾和争议**：当不同来源存在矛盾时，明确标注并说明各方观点

## 工作流

### 摄入（Ingest）

当用户将新资料放入 `raw/` 并要求处理时：

1. 阅读原始资料，与用户讨论关键要点
2. 在 `wiki/` 中创建来源摘要页
3. 识别资料中的概念、实体，检查 wiki 中是否已有对应页面：
   - 已有页面：更新内容，添加新来源的信息和引用
   - 无对应页面：创建新的概念页/实体页
4. 检查是否有主题页需要更新
5. 更新 `wiki/index.md`
6. 在 `wiki/log.md` 追加摄入记录
7. 向用户汇报：创建了哪些页面，更新了哪些页面，发现了什么值得注意的内容

### 查询（Query）

当用户提出问题时：

1. 先阅读 `wiki/index.md` 定位相关页面
2. 阅读相关 wiki 页面
3. 如需要，回溯到 `raw/` 中的原始资料获取更多细节
4. 综合回答，附带引用和页面链接
5. 如果回答具有长期价值（比较分析、综合论述等），建议将其归档为新的 wiki 页面

### 维护（Lint）

定期对 wiki 进行健康检查：

1. **矛盾检测**：不同页面间是否存在相互矛盾的说法
2. **过时信息**：是否有被新来源取代的旧说法
3. **孤立页面**：是否有没有任何入站链接的页面
4. **缺失页面**：是否有被频繁提到但尚未创建的概念/实体
5. **缺失交叉引用**：是否有应该互相链接但没有的页面
6. **信息缺口**：是否有可以通过网络搜索填补的空白
7. 生成维护报告，列出发现和建议

## 索引规则（index.md）

- 按页面类型分类组织
- 每个条目格式：`- [[页面名]] — 一句话描述`
- 每次摄入后更新
- 保持按类型分组、组内按字母/拼音排序

## 日志规则（log.md）

- 每个条目格式：`## [YYYY-MM-DD] 操作类型 | 标题`
- 操作类型：`摄入`、`查询`、`维护`、`创建`、`更新`
- 条目内容简要记录做了什么
- 新条目追加到文件顶部（最新的在前）
- 日志可通过 grep 解析：`grep "^## \[" wiki/log.md | head -5`
